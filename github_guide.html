Git & GitHub Workflow Guide

This guide provides a step-by-step reference for setting up Git, working with SSH keys, initializing repositories, and using GitHub effectively.

1. Set up Git Configuration

git config --global user.name "YourName"
git config --global user.email "your-email@example.com"
git config --list

This sets your username and email globally for all Git projects.

2. Create SSH Key

ssh-keygen -t rsa -b 4096

Follow the prompts and press Enter to accept the default location for the SSH key.

Add the SSH key to your SSH agent:

eval "$(ssh-agent -s)"
ssh-add ~/.ssh/id_rsa

3. Add SSH Key to GitHub

Copy the public key to your clipboard:

pbcopy < ~/.ssh/id_rsa.pub

Go to GitHub SSH and GPG Keys, click New SSH Key, paste your key, and click Add SSH Key.

4. Test SSH Connection

ssh -T git@github.com

If successful, you will see a message saying: You've successfully authenticated.

5. Initialize a Local Repository

cd /path/to/your/folder
git init

This initializes the folder as a local Git repository.

6. Track and Commit Files

git add .
git commit -m "Initial commit"

7. Link to a Remote Repository

git remote add origin git@github.com:username/repository.git

8. Push Changes to GitHub

git push -u origin main

This command pushes your local changes to the remote repository on GitHub.

9. Clone a Repository Using SSH

git clone git@github.com:username/repository.git

10. Summary of Common Commands

  • git status: Check the status of your repository.
  • git log: View the commit history.
  • git pull: Pull the latest changes from the remote.
  • git branch: List branches.
  • git checkout -b new-branch: Create and switch to a new branch.
